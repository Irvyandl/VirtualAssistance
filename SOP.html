Standard Operating Procedure

Purpose

Ensure an efficient, organized, and transparent collaboration between Irvin Prado (Virtual Assistant) and the client using Google Keep for task tracking and SOPs.

Scope

This SOP applies to all task requests, progress updates, completion reporting, and feedback exchange during the client’s active service plan.

1. Task Delegation – Client Responsibilities

  • Tasks must be submitted via Google Keep with the label "To Do – Current Week".
  • Include: Delegation Date & Time, Task Description, and Deadline.
  • Client priority color:
    • Peach or Red: Urgent
    • Sand or Yellow: Normal Priority
  • Notify via email/message when a task is delegated.

2. Task Acknowledgment – Assistant Responsibilities

  • Review task and reply with a recap within 30 minutes.
  • Log in a new note under label "Submitted Tasks".
  • Edit the original note: add submission footer and move to "Submitted Archive".
  • Assistant color:
    • Fog: Pending
    • Sand: In Progress
    • Mint: Completed

3. Task Completion

  • Update note with completion info, performance summary, and links.
  • Move note to "Done Tasks" and a copy to "Done Tasks Archive".
  • Notify the client via email once completed.

4. Weekly Maintenance

At the end of each week, all current task labels will be cleared except archives. A summary email will be sent to the client.

5. Feedback Process

  • Use a note labeled "Feedback" to share improvements.
  • Include: reference task, feedback, date submitted.
  • Feedback color:
    • Peach or Red: Urgent
    • Yellow: Non-urgent
  • Notify Irvin via email when feedback is added.

6. Format & Labels

All entries must use MM/DD/YYYY – 00:00 (Time Zone). This format is standard for all task tracking and archiving purposes.
