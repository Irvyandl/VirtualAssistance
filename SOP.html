Standard Operating Procedure

Purpose

Ensure an efficient, organized, and transparent collaboration between Irvin Prado (Virtual Assistant) and the client using Google Keep for task tracking and SOPs.

Scope

This SOP applies to all task requests, progress updates, completion reporting, and feedback exchange during the client’s active service plan.

1. Task Delegation – Client Responsibilities

  • Tasks must be submitted via Google Keep with the label "To Do – Current Week".
  • Inside the note, use the following format:
    • Task delegated on: [MM/DD/YYYY – 00:00 TZ]
    • Task description: [Details, links, expectations]
    • Completion target: [Today / Specific date / Any day this week]
  • Client priority color:
    • Peach or Red: Urgent
    • Sand or Yellow: Non-urgent
  • Send a brief notification (email or message) confirming task delegation.

2. Task Acknowledgment – Assistant Responsibilities

  • Review the note and reply with a summary recap within 30 minutes.
  • If no correction is received, create a new note under the label "Submitted Tasks" with the following format:
    • Task title
    • Original delegation date/time
    • Task description
    • Submission timestamp
    • Completion timestamp (to be added upon completion)
  • Edit the original note: add footer “Submitted to Irvin on [MM/DD/YYYY – 00:00 TZ]” and move it to "Submitted Tasks Archive".
  • Status background color in "Submitted Tasks":
    • Fog: Pending
    • Sand: In Progress
    • Mint: Completed

3. Task Completion

  • Update note with final time/date and performance summary.
  • Move to "Done Tasks" and copy to "Done Tasks Archive".
  • Send a brief email confirmation to the client.

4. Weekly Maintenance

At the end of each week, the following labels are cleared: To Do – Current Week, Submitted Tasks, and Done Tasks. Archives are kept for tracking. A summary email confirms weekly completions.

5. Feedback Process

  • Use label "Feedback" and format:
    • Task Reference: [Paste or link to original task]
    • Feedback: [Your suggestion]
    • Submitted on: [MM/DD/YYYY – 00:00 TZ]
  • Feedback priority:
    • Peach or Red: Urgent
    • Yellow: Non-urgent
  • Notify via message/email when feedback is submitted.

6. Format & Labels

All note timestamps and communications should follow this format: MM/DD/YYYY – 00:00 TZ. This ensures accurate archiving and easy reference for both parties.
